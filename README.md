# SMART-INFO-HUB

Автоматизированная платформа мониторинга простоев промышленного оборудования.

Логика обработки данных построена на стороне 1С, отказоустойчивое жизнеобеспечение и скрипты автоматизации — на Python.

 [Развертывание](#развертывание) | [Настройка 1С](#настройка-1с) | [Компоненты системы](#компоненты-системы)

Текстовый макет формы в интерфейсе `1С`:

```text
+------------------------------------------------------------------------------------------------------------------------------+
| Монитор простоев оборудования                                                                                               |
+------------------------------------------------------------------------------------------------------------------------------+
| Статус системы: Работает                                                                                                     |
| Последнее обновление: 26.04.2026 14:31                                                                                       |
| Текст ошибки:                                                                                                                |
+------------------------------------------------------------------------------------------------------------------------------+
| Дата фикс.       | Оборуд.      | Модель | Инв. №      | Причина                 | Исполнитель | План. оконч. рем. |
+------------------------------------------------------------------------------------------------------------------------------+
| 23.04.2026 08:15 | Пресс П6330  | П6330  | INV-0001    | Неисправность гидрос... | Мастер      | 23.04.2026 14:00 |
| 23.04.2026 09:40 | Станок 1К62  | 1К62   | INV-0002    | Замена электродвигат... | Мастер      | 23.04.2026 16:30 |
+------------------------------------------------------------------------------------------------------------------------------+
```

### Требования к структуре формы 1С
* **Табличная часть `ТаблицаМонитора`** (колонки): `ДатаФиксации`, `Оборудование`, `Модель`, `ИнвНомер`, `ПричинаПоломки`, `Исполнитель`, `ПлановаяДатаОкончанияРемонта`.
* **Реквизиты статуса:** `Статус_Системы`, `Последнее_Обновление`, `Текст_Ошибки` (выводятся на форму как *Статус системы*, *Последнее обновление*, *Текст ошибки*).

## Развертывание

Для локальной проверки логики обновления данных выполните в терминале:

```bash
python demo/demo_run.py
```

Если команда `python` недоступна в вашей конфигурации окружения, используйте:

```bash
py demo/demo_run.py
```

### Структура репозитория
* `src_1c/form_module.bsl` — Программный код модуля формы 1С.
* `src_1c/query_1c.txt` — Оптимизированный запрос выборки данных.
* `launcher/launcher.py` — Python-скрипт управления жизненным циклом.
* `assets/demo_screenshot.png` — Демонстрационные материалы интерфейса.

## Настройка 1С

1. Создать на форме реквизит `ТаблицаМонитора` типа `ТаблицаЗначений` и привязать к нему одноименное табличное поле.
2. Добавить статусные реквизиты формы: `Статус_Системы`, `Последнее_Обновление`, `Текст_Ошибки`.
3. Интегрировать исходный код из файла `src_1c/form_module.bsl` в модуль формы.
4. В теле запроса (`src_1c/query_1c.txt`) заменить `РегистрСведений.Задачи` на имя таблицы из вашей рабочей конфигурации.
5. Настроить и проверить обработчик ожидания автообновления формы (интервал — `60` секунд).

## Компоненты системы

### 1C Core
Внешняя обработка с автоматическим обновлением данных каждые `60` секунд. Реализует логику фильтрации по целевым группам ответственных лиц и номерам технических заданий.

### Python Launcher
Скрипт-автопилот для обеспечения отказоустойчивости и поддержания сессии `1С` в режиме `24/7`. Выполняет мониторинг целевого процесса с интервалом проверки в `5` минут и осуществляет автоматический перезапуск в случае сбоя.
